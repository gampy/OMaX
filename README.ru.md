## Обзор

OMaX — это надстройка для Microsoft Excel, которая позволяет строить отчёты, выполнять ad-hoc анализ и обновлять данные, работая напрямую с представлениями модели Optimacros. Инструмент ориентирован на специалистов финансово-учётной функции, привыкших работать в Microsoft Excel и регулярно взаимодействующих с данными Optimacros.

Возможности текущей версии:

- Строить снимки (срезы) Представлений модели Optimacros, переставлять (пивотировать) измерения, выполнять drill down и drill up по данным
- Редактировать данные и сохранять изменения на сервере
- Скрывать пустые строки отчёта
- Разворачивать иерархические справочники по уровням и работать с ними как с обычными измерениями в PivotTable

## Требования

| Компонент | Версия |
|---|---|
| Microsoft Excel (Windows, десктоп) | 2013+ |
| Optimacros | Любая версия с веб-сервисом Script endpoint |

> macOS Excel не поддерживается.

## Содержимое репозитория

| Файл | Описание |
|---|---|
| `OMaX.xlsm` | Клиентская рабочая книга |
| `REST API Gateway.js` | Скрипт Optimacros: валидация и диспетчеризация входящих HTTP-запросов |
| `REST API Data Service.js` | Скрипт Optimacros: чтение и запись данных мультикубов и справочников |

## Установка

### Серверная часть

В модели Optimacros откройте раздел **Скрипты** и создайте два скрипта из файлов репозитория:

- `REST API Gateway.js`
- `REST API Data Service.js`

В **Панели администратора -> API Services** создайте новый веб-сервис Endpoint и назначьте на него `REST API Gateway.js`.

### Клиентская часть

1. Откройте `OMaX.xlsm` в Excel. Разрешите макросы и активируйте Data Model / Power Pivot при запросе.
2. На листе **Connection** введите URL Endpoint и учётные данные Optimacros согласно комментариям в полях.
3. Нажмите **Подключиться** (Check Connection), чтобы проверить соединение.
4. При первом подключении к серверу Optimacros OMaX запросит разрешение на получение веб-контента по указанному Endpoint. Оставьте тип подключения **Анонимный** и нажмите **Подключиться**.
5. Нажмите **Загрузить модель** (Load Model) для загрузки Справочников (Lists) и Мультикубов (Multicubes).

## Использование

### DataPivot -> режим просмотра

| Кнопка | Действие |
|---|---|
| **Загрузить данные** | Загружает данные выбранного представления мультикуба из Optimacros |
| **Перестроить** | Строит или обновляет OLAP PivotTable на основе загруженных данных |
| **Сортировать** | Сортирует элементы справочников согласно их порядку в источнике Optimacros |
| **Редактировать** | Активирует режим редактирования |
| **Скрыть пустые / Показать пустые** | Переключает отображение пустых строк и столбцов PivotTable |
| **Показать формулы / Спрятать формулы** | Переключает видимость примечаний (формул кубов) в ячейках PivotTable |
| **Табличный вид / Структурный вид** | Переключает режим отображения сводной таблицы |

> Двойной клик по ячейке PivotTable также активирует режим редактирования.  
> В режиме редактирования лист заблокирован - изменение данных, фильтрация и сортировка недоступны.

### DataPivotEdit -> режим редактирования

| Кнопка | Действие |
|---|---|
| **Сохранить** | Сохраняет изменённые значения на сервере Optimacros |
| **Отменить** | Выходит из режима редактирования с подтверждением, отменяя несохранённые изменения |
| **Сбросить** | Перезагружает данные из DataPivot, отменяя все несохранённые изменения |

> Изменённые ячейки автоматически подсвечиваются через Условное Форматирование.
